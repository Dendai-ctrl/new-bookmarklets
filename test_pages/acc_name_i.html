Forms observations

  • AccName precedence: aria-labelledby then label for/id then title
    • Inspect and AViewer do not calculate AccName correctly
  • When title is provided with either aria-labelledby or label for/id, AccDescription=title.
  • **A-B (Gunderson) favelet requires mouse hover to reveal properites of form and the info disappears too quickly.
  • March 2016: In IE, A-B (Gunderson) favelets didn't markup local file content. (This may have only occurred on home PC.) June 9, 2016 update: works in IE and Chrome the same except for fieldset/legend.
  • **Inspect and AViewer show the same info, but AViewer is more difficult to use.
    • In IE, aria attributes are displayed separately instead of included in AccName or AccDescription. In Chrome, they are in AccName.
    • Tester must know to look for IAccessible2 when MSAA doesn't show full properties of the form.
    • The balloon info updates as it follows the cursor in IE; balloon must be manually toggled in Chrome to update.
  • **AViewer doesn't show fieldset/legend info
  • Visual Aria works in both IE and Chrome
    • Only marks up ARIA form fields on page (tool does not help tester find the non-ARIA forms)
    • Does not show fieldset/legend/label/ID properties on forms; otherwise shows same as A-B (Gunderson)
    • Hard to use in IE since markup is in black text and hard to find

Data Table observations

  • Data table cells, no difference in using aria-labelledby technique when compared to use of headers and id technique. JAWS read the cell data and headers the same in both techniques.
  • Use of the original TT ARIA favelet - IE: reveals only the labelledby assigned ID for the cells but no markup on the ID's so tester doesn't know if it's correct. Use WAT - Show Data tables first to mark ID's, then use ARIA favelet. Chrome: favelet markup adds cells and is confusing for tester. ID is shown on some, but not for all cells. If ID is shown, it is shown multiple times (once for each time it's called) and can be confusing.
  • No tool yet for coded aria data tables. Since there isn't an obvious reason to use ARIA on data tables, recommend using HTML techniques.

Images Observations

  • AccName precendence: aria-labelledby, aria-label, ALT then title. When both aria-describedby and aria-label provided, aria-label is ignored.
  • When both ALT and title provided, AccName=ALT and AccDescription=title.
  • AccDescription precedence if alt is present: aria-describedby then title. If only both of these are provided (see 7), title becomes accname. Otherwise, accname precedence and title will be ignored.
  • **A-B (Gunderson) favelet requires mouse hover to reveal properites of image.
  • ~March 2016: In IE, A-B (Gunderson) images favelet only marked the first image in above table. June 9, 2016 update: A-B works in IE the same as Chrome. No action taken from user side; fix must've been made elsewhere.
  • **AViewer is harder to read in IE when aria is present in IE. MSAA shows the accName, but accdescription is blank and tester must search the other sections for aria.
  • AViewer in Chrome only shows one set of info (MSAA or IA2 unknown) and displays the text from the referenced aria ID; in IE it shows the aria ID referenced (but not the text of the reference).
  • Visual ARIA only marks up ARIA and doesn't provide the AccName or AccDescription, only the ARIA ID.
